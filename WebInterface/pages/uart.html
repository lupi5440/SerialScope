UART (Universal Asynchronous Receiver-Transmitter)
¿Qué es UART?

UART es un protocolo de comunicación serial asíncrono que convierte datos paralelos en datos seriales para su transmisión. A diferencia de RS232 (que es un estándar de señalización eléctrica), UART es el circuito físico o periférico que implementa la comunicación serial asíncrona.

Implementación práctica UART

Diferencia entre UART y RS232:
UART
  • Es el circuito o periférico
  • Define el protocolo de comunicación
  • Niveles lógicos TTL/CMOS (0-5V o 0-3.3V)
  • Convierte paralelo a serial
  • Integrado en microcontroladores
RS232
  • Es el estándar eléctrico
  • Define los niveles de voltaje
  • Niveles ±3V a ±15V
  • Usa transceptor (ej: MAX232)
  • Para comunicación a mayor distancia
Nota importante: UART maneja el protocolo de comunicación serial, mientras que RS232 define los niveles de voltaje para transmisión. Un transceptor RS232 convierte las señales UART (TTL/CMOS) a niveles RS232 para comunicación a mayor distancia.
Características principales:
  • Asíncrono: No requiere señal de reloj compartida
  • Full-duplex: Transmisión y recepción simultáneas
  • Punto a punto: Comunicación directa entre dos dispositivos
  • Baud rate configurable: Velocidad acordada previamente
  • Frame personalizable: Bits de datos, paridad, stop bits
  • Niveles TTL/CMOS: Lógica digital estándar
Estructura del Frame UART:
IDLE (alto) → START bit (bajo) → DATA bits (5-9) → PARITY bit (opcional) → STOP bit(s) (1-2, alto) → IDLE
Campo Duración Descripción
IDLE Variable Línea en alto (1) cuando no hay transmisión
START bit 1 bit Transición de alto a bajo (1→0) indica inicio de trama
DATA bits 5-9 bits Datos a transmitir (típicamente 8 bits)
PARITY bit 0-1 bit Opcional: verificación de errores (par, impar, ninguno)
STOP bit(s) 1-2 bits Línea en alto (1) marca fin de trama
Configuraciones comunes de UART:
Configuración Data Bits Parity Stop Bits Uso
8N1 8 None (ninguna) 1 Más común en aplicaciones modernas
8N2 8 None 2 Mayor tolerancia a variaciones de timing
8E1 8 Even (par) 1 Con detección de errores
8O1 8 Odd (impar) 1 Con detección de errores
7E1 7 Even 1 ASCII de 7 bits con paridad
Velocidades de Baud Rate comunes:
Baud Rates Estándar:
  • 9600 - Más común, muy confiable
  • 19200 - Velocidad media
  • 38400 - Velocidad media-alta
  • 57600 - Alta velocidad
  • 115200 - Muy alta velocidad
Otros Baud Rates:
  • 300, 1200, 2400, 4800 - Velocidades lentas (legacy)
  • 230400, 460800, 921600 - Muy altas velocidades
  • 1000000, 2000000 - Ultra alta velocidad
Importante: Ambos dispositivos UART deben estar configurados con el mismo baud rate, número de bits de datos, paridad y bits de parada para comunicarse correctamente.
Conexiones UART:
Señal Nombre Dirección Descripción
TX Transmit Salida Transmite datos del dispositivo
RX Receive Entrada Recibe datos hacia el dispositivo
GND Ground - Referencia común de voltaje
Nota importante: TX de un dispositivo se conecta a RX del otro y viceversa. GND debe ser común entre ambos dispositivos.

Ventajas
  • Muy simple de implementar
  • Solo 2 líneas de datos (TX/RX) + GND
  • Full-duplex (transmisión simultánea)
  • No requiere reloj compartido
  • Ampliamente soportado
  • Bajo costo
  • Paridad opcional para detección de errores
Desventajas
  • Solo punto a punto (un maestro, un esclavo)
  • Requiere configuración previa idéntica
  • Sensible a errores de sincronización
  • Overhead de bits (START, STOP, PARITY)
  • Distancia limitada (sin RS232)
  • Velocidad limitada comparada con SPI
  • Sin control de flujo inherente
Aplicaciones comunes de UART:
Depuración: Consola serial, logs de sistema, debugging de microcontroladores
Módulos de comunicación: GPS, Bluetooth, WiFi, GSM/GPRS
Sensores: Módulos de sensores seriales (ultrasonido, CO2, etc.)
Comunicación entre MCUs: Arduino, ESP32, STM32, Raspberry Pi
Periféricos: Teclados, ratones, pantallas seriales
Industrial: Modbus RTU, instrumentación, control de equipos
Cálculo de tiempo de transmisión:

Fórmula:

Tiempo por bit = 1 / Baud Rate
Tiempo por frame = (START + DATA + PARITY + STOP) / Baud Rate

Ejemplo con configuración 8N1 a 9600 baudios:

  • Tiempo por bit = 1 / 9600 = 104.16 µs
  • Bits por frame = 1 (START) + 8 (DATA) + 0 (PARITY) + 1 (STOP) = 10 bits
  • Tiempo por frame = 10 × 104.16 µs = 1.041 ms
  • Bytes por segundo ≈ 960 bytes/s (9600 / 10)
