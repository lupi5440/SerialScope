Fundamentos de comunicación

Comunicación Serial vs Paralela
Comunicación Serial

En la comunicación serial, los datos se transmiten bit por bit a través de un solo canal o línea de comunicación.

Ventajas:
  • Menos cables requeridos
  • Menor costo de implementación
  • Ideal para largas distancias
  • Menos interferencia electromagnética
  • Más fácil de expandir
Desventajas:
  • Menor velocidad de transferencia
  • Más compleja de sincronizar
Ejemplos: I2C, SPI, USB, RS-232
Comunicación Paralela

En la comunicación paralela, múltiples bits se transmiten simultáneamente a través de múltiples canales o líneas.

Ventajas:
  • Mayor velocidad de transferencia
  • Transmisión simultánea de datos
  • Más simple a cortas distancias
Desventajas:
  • Muchos cables requeridos
  • Mayor costo
  • Problemas de sincronización
  • No apta para largas distancias
  • Mayor interferencia electromagnética
Ejemplos: Bus de datos de CPU, Puerto paralelo (obsoleto), RAM
Comunicación Síncrona vs Asíncrona
Comunicación Síncrona

La comunicación síncrona utiliza una señal de reloj compartida entre el transmisor y el receptor para sincronizar la transferencia de datos.

Características:
  • Requiere señal de reloj (CLK)
  • Sincronización precisa
  • Mayor velocidad de transferencia
  • Transmisión continua de datos
  • Más compleja de implementar
Ventajas:
  • Mayor eficiencia en transmisión
  • Menor overhead de bits
  • Velocidades más altas
Desventajas:
  • Requiere línea adicional de reloj
  • Ambos dispositivos deben estar sincronizados
Ejemplos: I2C, SPI
Comunicación Asíncrona

La comunicación asíncrona no requiere señal de reloj compartida. Los dispositivos deben acordar previamente la velocidad de comunicación.

Características:
  • No requiere señal de reloj
  • Usa bits de inicio/parada
  • Menor velocidad relativa
  • Velocidad preconfigurada (baudios)
  • Más simple de implementar
Ventajas:
  • Menos cables requeridos
  • Implementación más simple
  • Flexible en cuanto a timing
Desventajas:
  • Mayor overhead (bits de control)
  • Velocidades más limitadas
  • Posible desincronización
Ejemplos: RS-232 / UART
¿Para qué sirven los protocolos de comunicación?

Los protocolos de comunicación son conjuntos de reglas y estándares que permiten la transmisión confiable de datos entre dispositivos electrónicos.

Funciones principales:
  • Estandarización: Establecen reglas comunes para que diferentes dispositivos puedan comunicarse sin importar el fabricante.
  • Control de errores: Detectan y corrigen errores en la transmisión de datos mediante bits de paridad, checksums o CRC.
  • Sincronización: Aseguran que el transmisor y receptor estén coordinados en tiempo y velocidad.
  • Direccionamiento: Permiten identificar el destinatario correcto en sistemas multi-dispositivo.
  • Control de flujo: Regulan la velocidad de transmisión para evitar saturación del receptor.
Aplicaciones prácticas:
  • IoT y Sensores: Comunicación entre microcontroladores y sensores (temperatura, humedad, presión).
  • Almacenamiento: Comunicación con memorias EEPROM, tarjetas SD, memorias flash.
  • Periféricos: Conexión de displays, teclados, módulos GPS, Bluetooth.
  • Industrial: Control de maquinaria, sistemas SCADA, PLCs.
  • Automotriz: Sistemas de diagnóstico, sensores vehiculares, CAN bus.
¿Por qué son importantes?

Sin protocolos estandarizados, cada fabricante tendría su propio método de comunicación, haciendo imposible la interoperabilidad entre dispositivos. Los protocolos garantizan que un sensor de un fabricante pueda comunicarse con un microcontrolador de otro fabricante, siempre que ambos cumplan con el mismo estándar.
