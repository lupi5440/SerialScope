Recommended Standard 232 - Comunicación Serial Asíncrona

RS232 (Recommended Standard 232)
¿Qué es RS232?

RS232 es un estándar de comunicación asíncrono serial que utiliza dos líneas (TX para transmitir y RX para recibir) y se basa en la velocidad de baudios previamente acordada entre dispositivos. Es ampliamente usado en interfaces de comunicación de computadoras.

Introducción al RS232

RS232 estándar de comunicación

Características principales:
  • Asíncrono: No requiere señal de reloj compartida
  • Punto a punto: Comunicación entre dos dispositivos
  • Baudios configurables: 9600, 115200, etc.
  • Bits de inicio y parada: Sincronización de trama
  • Paridad opcional: Detección de errores
  • Niveles de voltaje: ±3V a ±15V (típicamente ±12V)
  • Distancia máxima: Hasta 15 metros a baja velocidad
Estructura de la trama:
START (1 bit) → DATA (5-9 bits) → PARITY (0-1 bit) → STOP (1-2 bits)
Configuraciones típicas:
  • 8N1: 8 bits de datos, sin paridad, 1 bit de parada (más común)
  • 7E1: 7 bits de datos, paridad par, 1 bit de parada
  • 8E1: 8 bits de datos, paridad par, 1 bit de parada
Señales principales:
Señal Nombre Descripción
TX Transmit Data Datos de transmisión
RX Receive Data Datos de recepción
RTS Request To Send Solicitud para enviar
CTS Clear To Send Listo para enviar
DTR Data Terminal Ready Terminal de datos listo
DSR Data Set Ready Conjunto de datos listo
Conectores Físicos:

El estándar RS232 define dos tipos principales de conectores:

DB-9 (DE-9) - 9 pines

Conector DB-9

Uso: Más común en computadoras modernas y aplicaciones básicas.

Ventajas:

  • Más compacto y fácil de manejar
  • Suficiente para la mayoría de aplicaciones
  • Estándar en PCs desde los años 90

Configuración de pines:

Pin Señal Función
1 DCD Data Carrier Detect (Detección de portadora)
2 RX Receive Data (Recepción de datos)
3 TX Transmit Data (Transmisión de datos)
4 DTR Data Terminal Ready (Terminal listo)
5 GND Signal Ground (Tierra de señal)
6 DSR Data Set Ready (Conjunto de datos listo)
7 RTS Request To Send (Solicitud para enviar)
8 CTS Clear To Send (Listo para enviar)
9 RI Ring Indicator (Indicador de llamada)
DB-25 - 25 pines

Conector DB-25

Uso: Estándar original, usado en equipos antiguos e industriales.

Ventajas:

  • Soporta todas las señales del estándar RS232
  • Permite control completo de hardware
  • Común en equipos industriales legacy

Configuración de pines (principales):

Pin Señal Función
1 FG Frame Ground (Tierra del chasis)
2 TX Transmit Data (Transmisión de datos)
3 RX Receive Data (Recepción de datos)
4 RTS Request To Send (Solicitud para enviar)
5 CTS Clear To Send (Listo para enviar)
6 DSR Data Set Ready (Conjunto de datos listo)
7 GND Signal Ground (Tierra de señal)
8 DCD Data Carrier Detect (Detección de portadora)
20 DTR Data Terminal Ready (Terminal listo)
22 RI Ring Indicator (Indicador de llamada)
Pines 9-19, 21, 23-25: Señales secundarias y de sincronización
¿Por qué existen dos versiones?

El DB-25 fue el conector original definido en el estándar RS232 en 1969, con 25 pines para soportar todas las señales de control posibles. Sin embargo, en la práctica, la mayoría de aplicaciones solo necesitan 9 señales básicas.

En los años 80, IBM introdujo el conector DB-9 más compacto en sus computadoras personales, eliminando las señales raramente usadas. Esto se convirtió en el estándar de facto para PCs, aunque el DB-25 sigue siendo común en equipos industriales que requieren control completo de hardware.

Aplicaciones comunes:
Comunicación con PC: Terminales seriales, consolas de depuración
Equipos industriales: PLC, instrumentación, control de procesos
Dispositivos embebidos: Microcontroladores, módems, GPS
Automatización: Sistemas SCADA, control de máquinas
